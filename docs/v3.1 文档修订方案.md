## v3.1 文档修订方案（保留愿景，但按 Current/Next/Target 拆层）

### Summary
采用你选的“保留愿景但拆层”策略：不删方向，删“当前已具备”的过度承诺。所有“当前能力”必须能在代码和测试中找到证据，否则降级为 Next/Target，并补前置条件与验收口径。

### Key Changes
1. 把“当前范围内含 Reconciler/AI 研究辅助”降级到 Next/Target，并在文档中加状态表（Current/Next/Target）。
现状证据：服务层和路由层无 Reconciler/AI 模块，[services/__init__.py](/d:/Code%20of%20Python/quant_trading_system/trader/services/__init__.py#L6)、[routes/__init__.py](/d:/Code%20of%20Python/quant_trading_system/trader/api/routes/__init__.py#L6)。

2. 把“Alignment Gate 停止外发正式市场事件”改为“按流域生效”。
建议文案：Private/Execution 事件在 Gate 期间阻断，Public 行情流可继续但必须打 DEGRADED 标签。
现状证据：Private 在 ALIGNING 丢弃消息，[private_stream.py](/d:/Code%20of%20Python/quant_trading_system/trader/adapters/binance/private_stream.py#L394)；Public 仍继续外发，[public_stream.py](/d:/Code%20of%20Python/quant_trading_system/trader/adapters/binance/public_stream.py#L233)、[connector.py](/d:/Code%20of%20Python/quant_trading_system/trader/adapters/binance/connector.py#L211)。

3. 把“PostgreSQL 为当前正式持久化底座”改成“Risk 事务链路已 PG-First，其它读模型仍内存”。
现状证据：风险仓库支持 PG 优先+内存回退，[risk_repository.py](/d:/Code%20of%20Python/quant_trading_system/trader/adapters/persistence/risk_repository.py#L7)；事件查询服务仍走内存存储，[event.py](/d:/Code%20of%20Python/quant_trading_system/trader/services/event.py#L13)、[in_memory.py](/d:/Code%20of%20Python/quant_trading_system/trader/storage/in_memory.py#L12)。

4. 把“Layer A 唯一允许直接驱动执行”改成条件句。
建议文案：仅在 Runner/Execution 主链路上线并通过 Gate 后，Layer A 才成为唯一执行驱动源；当前阶段为数据治理与对齐主源，不直接自动下单。

5. 把“8 周闭环”改成“Gate 驱动里程碑”，保留 Sprint 名称但取消硬周数承诺。
建议新增：每个 Sprint 的 Entry/Exit Gate（代码存在性 + 非跳过测试 + 故障演练）三项同时满足才算完成。

6. 统一 KillSwitch 级别语义，避免文档与实现冲突。
现状证据：适配器侧 L2/L3 命名与风险引擎语义不一致，[environmental_risk.py](/d:/Code%20of%20Python/quant_trading_system/trader/adapters/binance/environmental_risk.py#L39)、[risk_engine.py](/d:/Code%20of%20Python/quant_trading_system/trader/core/application/risk_engine.py#L42)。
建议新增“Canonical Level Map（数字为准，别名兼容）”。

7. 把“A 股适配器占位”从“类实现占位”改成“接口契约占位”。
建议文案：当前只要求 BrokerPort/MarketDataPort 契约稳定，不要求落地 ChinaBrokerAdapter 类实体。

### Public APIs / Types 声明修订
1. 将 `/v1/events`、`/v1/snapshots/latest` 的“真理源”声明改为“当前内存读模型，目标切 PG 投影读模型”。
2. 将 AI 相关接口（如 proposal/approve）统一标注为 Target，未落地前不得写成 Current。
3. 在文档新增 `Capability Matrix`：每个接口标注 `Current | Next | Target` 与证据链接（代码路径 + 测试名）。

### Test Plan（文档验收）
1. 文档 lint：所有“Current”条目必须附 1 个代码路径 + 1 个非跳过测试名。
2. 差距校验：搜索 Reconciler/AI endpoint，若不存在则文档不得标 Current。
3. 持久化校验：若声明 PG 为 Current，需同时满足“服务读链路已接 PG”与“PG 测试非 skip”。
4. 回归校验：继续跑代表性测试集（你当前环境可跑通的子集），并记录 skip 原因（如 PG 未配置）避免误报完成。

### Assumptions
1. 本轮只改文档定位，不同步要求立即补齐代码。
2. 版本策略采用“小版本修订”（建议 v3.1.1），仅修正能力分层与验收口径，不改总体愿景。
3. Current 的判定标准固定为“代码存在 + 测试可证 + 运行前提明确”。
