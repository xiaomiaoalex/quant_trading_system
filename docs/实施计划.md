# quant_trading_system v2 架构规范完善与实施计划（可直接执行）

## 摘要
本计划基于已完成的代码遍历与测试抽样，目标是在“功能并行推进”的前提下先清除生产阻塞缺陷，并将主文档升级为可执行规范 v2。  
当前已确认的关键阻塞包括：连接器初始化失败、确定性层死锁/重置缺陷、私有流解析不稳、事件类型不一致、控制面缺少 `/v1/risk/events` 最小闭环接口。  
首发范围锁定为 Binance 单账户，严格 Fail-Closed，事件存储直上 PostgreSQL，部署采用单机 Docker Compose。

## 现状基线（已验证）
1. 通过：`test_architecture.py`、`test_api_endpoints.py`、`test_binance_degraded_cascade.py`、`test_binance_rest_alignment.py`。  
2. 失败：`test_binance_connector.py`（12项失败，`PrivateStreamConfig(testnet=...)` 参数不匹配）。  
3. 失败：`test_binance_private_stream.py`（`trade_id` 解析对字符串不兼容）。  
4. 失败：`test_deterministic_layer.py::TestDeterministicApplier::test_reset`（`PendingBuffer` 无 `clear()`）。  
5. 超时：`test_deterministic_layer.py` 全量（`TTLSet.__len__` 可重入锁死风险）。  
6. 运行时缺陷：`compute_exec_key` 缺少 `hashlib` 导入；`create_position_updated_event` 使用不存在的 `EventType.POSITION_UPDATED`。
